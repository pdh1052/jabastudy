익명의 내부 클래스

클래스는 구조적으로 "class"키워드가 있어야 클래스이다.
익명의 내부클래스는 class키워드가 없기때문에 "익명"의 내부클래스.

2가지
변수를 선언하면 쓰는 익명의 내부 클래스 : 가짜 익명의 내부클래스
변수 선언 없이 사용하는 익명의 내부 클래스 : 진짜  " " "


=========================================


try ~~catch : 개발자의 잘못을 처리하기 위해 사용해서는 안 됨.
>>>>>>>>>예외가 발생했어도 예외가 없는 것과 같은 효과.
try : 예외가 발생할 수 있는 코드 
catch : 예외가 발생할 때 실행할 코드
-문법적으로 이상이 없지만, 논리적으로 이상이 있을 경우 사용
-사용자로부터 데이터를 입력받는데,
 사용자가 잘못된 데이터를 입력했을 때 대비해서 사용.

throws : 예외가 발생했을 때, 해당 메서드를 호출한 곳에서
           예외처리를 하도록 예외를 던져줌.

finally : 예외가 발생하든 그렇지 않든 항상 실행해야하는 코드는
          finally에 구현. return보다 더 강력함.

throw : 문법적으로는 예외는 아니지만 논리적인 예외를 발생하는 기능

사용자 정의 예외 클래스 : 
가능하면 Exception을 상속하도록하여, throws를 날리도록 구현하고,
그러면 호출하는 메서드에서 try~catch작업이 가능하니까.
